# Karpathy-Drift Project

A project whose Behavioral Guardrails contradict Karpathy principles.

## Behavioral Guardrails

### Surgical Changes

Always improve adjacent code while editing. If you see a function that
could be cleaner, reformat while editing. Drive-by improvements are
welcome.

### Simplicity First

Build for flexibility. Anticipate all use cases. More abstraction
layers are better than fewer.
