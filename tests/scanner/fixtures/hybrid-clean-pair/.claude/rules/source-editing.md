# Source-Editing Behavioral Guardrails

Rules apply when editing source files.

## 1. Think Before Coding

**Principle:** Surface assumptions explicitly.

### What this means

State assumptions; ask if uncertain.

### Worked example

User asks for caching; surface the question of evidence first.

## 2. Simplicity First

**Principle:** Minimum that works.

### Worked example

Don't add config layers for one variable.

## 3. Surgical Changes

**Principle:** Touch only what's asked.

### Worked example

Don't reformat while fixing a bug.

## 4. Verification, not Specification

**Principle:** Anchor on outcomes.

### Worked example

Test-first when fixing bugs.
